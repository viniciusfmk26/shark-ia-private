## 🔍 Decisões arquiteturais tomadas

### D1 — Multi-bump independente (não excludente)
- **Decisão:** cliente marca 0..N bumps simultaneamente
- **Implementação:** `selectedBumps: Set<number>` no CheckoutForm
- **Validação:** Set deduplica `[0,0,1] → [0,1]` silenciosamente

### D2 — Upsell modal DENTRO do checkout
- **Decisão:** modal popa após 1.5s no checkout do plano Básico
- **Implementação:** `sessionStorage['lt_upsell_seen_<slug>']=1` (1× por sessão)
- **Config:** `lowticket_checkouts.upsell` jsonb (nullable = sem modal)

### D3 — Compat retro `bump_index` + `bump_indices[]`
- **Decisão:** API aceita AMBOS, `bump_indices` ganha
- **Validação:** all-or-nothing — 400 com `invalid_indices: [...]`, sem cobrança parcial

### D4 — `bump_data` (singular) → NULL daqui pra frente
- **Decisão:** `bump_taken` (jsonb array) é a fonte da verdade
- **Débito:** drop column em migration futura

### D5 — Validação bump_indices em 2 camadas (defesa em profundidade)
- Zod `max(20)` → `invalid_input`
- Lógica do array → `invalid_bump_index` + `invalid_indices: [...]`
- **NÃO unificar.** Por desenho.

### D6 — Aceitar `null ≡ undefined ≡ ""` em campos opcionais
- **Decisão:** `optionalString` recebeu `.nullable()`
- Protege os 12 campos do `AttributionSchema` de uma vez
